# Phase 0 ŌĆö Episode plan migration preflight (repo)

This document is a **repo-only** preflight. Execute the SQL blocks against production or a recent staging clone before Phase 1 cutover, and attach results to the go/no-go record.

## Runner semantics (authoritative)

- `scripts/run-all-migrations.js` runs **one** `pool.query(sql)` per migration file (entire file as a single statement batch).
- There is **no** implicit outer transaction unless the file contains `BEGIN`/`COMMIT`.
- Do **not** use `CREATE INDEX CONCURRENTLY` inside a migration file that runs inside a transaction; use a separate operator step if `CONCURRENTLY` is required.

## Database ŌĆö appointments triggers (run in target DB)

```sql
SELECT tgname, tgenabled
FROM pg_trigger
WHERE tgrelid = 'appointments'::regclass
  AND NOT tgisinternal;
```

Document each trigger name, purpose, and whether new nullable columns (`plan_item_id`, etc.) are safe.

## Phase 1 block conditions (from plan)

Do not start Phase 1 DDL in prod while:

- Any user trigger on `appointments` is undocumented vs. the query above.
- A consumer is sensitive to new nullable columns without verification.
- `CREATE INDEX CONCURRENTLY` is required but no operator path exists.
