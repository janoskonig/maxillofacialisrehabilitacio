## Non-transactional / operator-only (when needed)

- Large indexes: `CREATE INDEX CONCURRENTLY` in a **standalone** `psql` session (not inside `BEGIN`), then document in `node_migrations` manually only if you adopt a split process — default migrations here stay transactional.

## Lock order (Phase 3)

1. `SELECT … FOR UPDATE OF a SKIP LOCKED`
2. `UPDATE` only selected ids
3. Short transactions (`BATCH_SIZE`, `MAX_BATCHES`)
